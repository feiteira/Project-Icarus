# Memory - April 17, 2026

## MediaWiki Infobox Fix - Critical Syntax Discovery

The MediaWiki infobox tables were "missing TRs" — all trivia data appeared in a single table row. **Root cause**: MediaWiki table syntax requires `|-` to start a new row. `|` on separate lines does NOT create new rows — it continues adding cells to the current row.

**Correct pattern** (each trivia = its own row):
```
|- style="background:#f0f0f0;"
| Label || Value
```

**Wrong pattern** (all cells in one row):
```
| Label
| Value
```

The rebuild script had a detection bug: it scanned wikitext line-by-line looking for `|` separators and counted cells. When it found 8+ cells (image+caption+4 trivia pairs), it reported "OK" — but the actual wikitext only had `|` on separate lines without `|-` between them, so MediaWiki rendered everything as ONE row.

**Pages that needed fixing**: Cold War, United States, Roman Empire, Holy Roman Empire, Federal Reserve, World Economic Forum, Silk Road, Saudi Arabia, Game Theory.

**Fix method**: Direct SQLite update — modify text blob, update content_size and content_sha1, keep content_address pointing to same old_id (doesn't create new revision but updates the slot's content).

**Verification**: Check rendered HTML for `<tr>` count in infobox — should match number of infobox rows (1 image + N trivia).
